### 半固定层（内容按实际填充）

- **规范目的**：固定一句话，内容根据规范主题填充
- **规则列表**：每条规则必须包含：
  - 规则编号（递增）
  - 强度标注（⛔ MUST / ⚠️ SHOULD / 💡 MAY）
  - 规则描述
  - ✅ 正确示例（Java 代码）
  - ❌ 错误示例（Java 代码）
- **检查清单**：每条规则对应一个 checkbox 检查项，用于 Code Review
- **相关文档**：链接到其他规范或模块文档

#### 可选扩展内容

以下内容为可选扩展，根据规则的重要性和复杂度选择性添加：

- **💡 设计理由**（紧跟规则之后，1-3 句）：说明为什么这样规定而非其他方案。适用场景：⛔ MUST 级别规则、与业界常见做法不同的规则。格式示例：
  > **为什么**：`@Data` 含 `@EqualsAndHashCode` 和 `@ToString`，在 Entity 继承链中会导致栈溢出或意外行为。

- **⚠️ 常见违规场景**（2+ 个场景）：列出实际项目中容易犯的错误，每个场景包含 ❌ 错误写法和 ✅ 正确写法。适用场景：⛔ MUST 级别规则、新人容易犯错的规则。格式示例：
  > **常见违规**：
  > 1. **在 Controller 中注入 Mapper** — ❌ 应通过 Service/Facade 访问 → ✅ 使用对应的 Service 接口

- **🔗 关联规则**（1-2 条链接）：说明本规则与其他规则的关系（前置规则、互补规则、例外规则）。适用场景：规则之间有依赖关系时。格式示例：
  > **关联规则**：本规则是 [规则 1: 四层架构约束](#规则-1-四层架构约束) 的具体体现
